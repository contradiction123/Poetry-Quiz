# 编码问题说明

## 问题原因

在Windows系统中，当使用PyInstaller打包包含中文文件名的程序时，可能会遇到编码问题导致文件名乱码。

### 根本原因

1. **Windows命令行编码**：默认使用GBK/CP936编码
2. **Python文件编码**：通常使用UTF-8编码
3. **PyInstaller处理**：在处理中文文件名时可能出现编码转换问题

## 解决方案

### ✅ 推荐方案：使用英文文件名

1. **打包时使用英文名称**
   - 使用 `poetry_answer_tool.spec` 文件
   - 生成的文件名：`PoetryAnswerTool.exe`

2. **打包后手动重命名**（可选）
   - 在Windows资源管理器中右键重命名
   - 改为：`诗词答题工具.exe`
   - 这样在资源管理器中显示正常，不会影响程序运行

### 当前项目配置

- **Spec文件**：`poetry_answer_tool.spec`（UTF-8编码，英文名称）
- **输出文件**：`PoetryAnswerTool.exe`（英文名称，无编码问题）
- **配置文件**：`config.json`（UTF-8编码）

## 文件说明

### 正确的文件
- ✅ `poetry_answer_tool.spec` - 打包配置文件（使用这个）
- ✅ `PoetryAnswerTool.exe` - 可执行文件（在dist目录）

### 如果看到乱码文件
如果看到类似 `璇楄瘝绛旈宸ュ叿.spec` 这样的乱码文件：
1. 这些是旧的打包文件，可以删除
2. 运行 `cleanup.bat` 脚本自动清理
3. 或者手动删除这些乱码文件

## 打包命令

```bash
# 使用正确的spec文件打包
pyinstaller poetry_answer_tool.spec --clean

# 或使用批处理脚本
build.bat
```

## 编码设置

### Spec文件编码
```python
# -*- mode: python ; coding: utf-8 -*-
```
文件开头必须声明UTF-8编码

### 命令行编码
```batch
chcp 65001  # 设置UTF-8编码
```

## 验证方法

打包完成后检查：
1. ✅ `dist` 目录中有 `PoetryAnswerTool.exe`
2. ✅ 文件名显示正常（无乱码）
3. ✅ 文件可以正常运行
4. ✅ 如果重命名为中文，在资源管理器中显示正常

## 注意事项

1. **不要使用中文文件名打包**：会导致编码问题
2. **使用spec文件**：可以更好地控制打包过程
3. **清理旧文件**：定期清理build和dist目录中的旧文件
4. **重命名时机**：在打包完成后手动重命名，而不是在打包时使用中文

## 清理脚本

运行 `cleanup.bat` 可以自动清理所有乱码文件。
