# 诗词答题自动化工具 - 执行步骤

## 环境准备

### 运行环境

#### 1. Python环境
- **Python版本**：Python 3.8 或更高版本（推荐 3.9+）
- **下载地址**：https://www.python.org/downloads/
- **安装注意事项**：
  - 安装时勾选 "Add Python to PATH"
  - 选择 "Install for all users"（可选）

#### 2. 系统要求
- **操作系统**：Windows 10/11
- **内存**：至少 4GB RAM（推荐 8GB+）
- **磁盘空间**：至少 2GB 可用空间（用于安装依赖和模型文件）
- **网络**：需要网络连接（首次运行下载OCR模型，运行时调用AI API）

#### 3. 雷电模拟器
- 安装并运行雷电模拟器
- 确保模拟器窗口可见且不被遮挡
- 建议设置模拟器分辨率为 1920x1080 或更高

### 步骤14：打包为可执行文件

#### 14.1 安装PyInstaller

```bash
pip install pyinstaller
```

#### 14.2 创建打包脚本

创建 `build.bat` 文件（Windows批处理脚本）：

```batch
@echo off
echo 开始打包程序...

pyinstaller --name="诗词答题工具" ^
    --onefile ^
    --windowed ^
    --icon=NONE ^
    --add-data="config.json;." ^
    --hidden-import=paddleocr ^
    --hidden-import=mss ^
    --hidden-import=PIL ^
    --hidden-import=pyautogui ^
    --hidden-import=requests ^
    main.py

echo 打包完成！
echo 可执行文件位置: dist\诗词答题工具.exe
pause
```

**参数说明**：
- `--name`：生成的exe文件名
- `--onefile`：打包成单个exe文件
- `--windowed`：不显示控制台窗口（GUI程序）
- `--icon`：图标文件路径（如果有）
- `--add-data`：添加数据文件（配置文件等）
- `--hidden-import`：显式导入隐藏的模块

#### 14.3 执行打包

1. 运行打包脚本
   ```bash
   build.bat
   ```
   
   或者直接使用命令：
   ```bash
   pyinstaller --name="诗词答题工具" --onefile --windowed --add-data="config.json;." main.py
   ```

2. 等待打包完成
   - 打包过程可能需要几分钟
   - 首次打包会下载一些依赖

3. 检查输出
   - 打包完成后，exe文件在 `dist` 目录下
   - 文件名为 `诗词答题工具.exe`

#### 14.4 打包优化（可选）

如果打包后的exe文件太大，可以：

1. **使用UPX压缩**（可选）
   ```bash
   pyinstaller --upx-dir=path/to/upx --onefile --windowed main.py
   ```

2. **排除不需要的模块**
   ```bash
   pyinstaller --exclude-module=matplotlib --exclude-module=numpy --onefile main.py
   ```

## 常见问题解决

### 问题1：PaddleOCR安装失败

**解决方案**：
```bash
# 使用国内镜像
pip install paddlepaddle -i https://pypi.tuna.tsinghua.edu.cn/simple
pip install paddleocr -i https://pypi.tuna.tsinghua.edu.cn/simple
```

### 问题2：打包后exe文件太大

**解决方案**：
- 使用虚拟环境，只安装必要的包
- 使用 `--exclude-module` 排除不需要的模块
- 考虑使用 `--onedir` 而不是 `--onefile`

### 问题3：打包后无法运行

**解决方案**：
- 使用 `--console` 参数查看错误信息
- 检查是否缺少数据文件（config.json等）
- 检查是否有隐藏导入的模块

### 问题4：OCR识别速度慢

**解决方案**：
- 只识别关键区域，不识别全屏
- 使用GPU加速（如果有NVIDIA显卡）
- 优化OCR参数
