# 打包说明

## 编码问题解决方案

### 问题描述
在Windows系统中，PyInstaller打包时如果使用中文文件名，可能会因为编码问题导致文件名乱码。

### 解决方案

#### 方案1：使用英文文件名（推荐）
- 打包时使用英文名称：`PoetryAnswerTool.exe`
- 打包完成后，可以手动重命名为中文名称
- 这是最稳定的方案

#### 方案2：使用UTF-8编码
如果必须使用中文文件名，需要：
1. 确保Python文件使用UTF-8编码保存
2. 在命令行中设置UTF-8编码：
   ```batch
   chcp 65001
   ```
3. 使用spec文件指定UTF-8编码：
   ```python
   # -*- mode: python ; coding: utf-8 -*-
   ```

### 当前打包配置

我们使用 `poetry_answer_tool.spec` 文件进行打包：
- 文件名：`PoetryAnswerTool.exe`（英文，避免编码问题）
- 编码：UTF-8
- 包含所有必要的模块和依赖

### 打包命令

```bash
# 使用spec文件打包（推荐）
pyinstaller poetry_answer_tool.spec --clean

# 或使用批处理脚本
build.bat
```

### 重命名建议

打包完成后，如果需要中文名称，可以：
1. 手动重命名 `PoetryAnswerTool.exe` 为 `诗词答题工具.exe`
2. 确保重命名后的文件名在Windows资源管理器中显示正常

### 模块导入问题

如果遇到 `No module named 'xxx'` 错误：
1. 检查 `poetry_answer_tool.spec` 文件中的 `hiddenimports` 列表
2. 确保所有本地模块都被添加到列表中：
   - `ocr_engine`
   - `screen_capture`
   - `question_parser`
   - `ai_client`
   - `click_handler`
   - `config`
   - `database`
   - `ai_providers`
   - `ai_providers.deepseek`

### 验证打包

打包完成后，检查：
1. `dist` 目录中是否有 `PoetryAnswerTool.exe`
2. 文件大小是否正常（约100-200MB，包含所有依赖）
3. 运行exe文件，检查是否能正常启动
