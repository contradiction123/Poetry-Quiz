# 打包分发指南

本指南说明如何将程序打包成可执行文件，分发给没有Python环境的用户使用。

## 一、打包步骤

### 1. 准备工作

确保你已经：
- 安装了Python 3.8或更高版本
- 创建了虚拟环境并安装了所有依赖

### 2. 安装打包工具

如果还没有安装PyInstaller，运行：

```bash
pip install pyinstaller>=5.13.0
```

或者直接安装所有依赖：

```bash
pip install -r requirements.txt
```

### 3. 执行打包

**方法1：使用批处理脚本（推荐）**

直接双击运行 `build.bat` 文件，或在命令行中执行：

```bash
build.bat
```

**方法2：手动打包**

```bash
# 激活虚拟环境
venv\Scripts\activate

# 执行打包
pyinstaller poetry_answer_tool.spec --clean
```

### 4. 打包结果

打包完成后，可执行文件位于 `dist\PoetryAnswerTool.exe`

## 二、打包文件说明

### 文件大小

打包后的exe文件大小约为：
- **基础版本**：100-200MB（不包含OCR模型）
- **完整版本**：200-500MB（包含OCR模型，如果打包时已下载）

**注意**：首次运行时，如果OCR模型未包含在exe中，程序会自动下载（约100MB），需要网络连接。

### 打包包含的内容

- Python解释器
- 所有Python依赖库
- 程序源代码（已编译）
- 配置文件模板

### 不包含的内容

- OCR模型文件（首次运行自动下载）
- 用户配置文件（config.json，需要用户自己创建）
- 数据库文件（运行时自动创建）

## 三、分发给其他用户

### 需要分发的文件

1. **PoetryAnswerTool.exe** - 主程序文件
2. **config.example.json** - 配置文件模板（用户需要重命名为config.json并配置API Key）

### 分发步骤

1. 将 `dist\PoetryAnswerTool.exe` 复制到新文件夹
2. 将 `config.example.json` 复制到同一文件夹
3. 创建一个 `使用说明.txt` 文件（见下方模板）
4. 将整个文件夹压缩后分发给用户

### 用户使用说明模板

创建 `使用说明.txt` 文件，内容如下：


## 四、优化打包大小（可选）

如果打包后的文件过大，可以尝试以下优化：

### 1. 排除不必要的模块

在 `poetry_answer_tool.spec` 文件的 `excludes` 列表中添加不需要的模块。

### 2. 使用UPX压缩

UPX可以压缩exe文件，但可能被杀毒软件误报。如果使用，需要：
- 下载UPX工具
- 在spec文件中设置 `upx=True`

### 3. 分离OCR模型

不将OCR模型打包进exe，让程序首次运行时自动下载（当前默认方式）。

## 五、测试打包结果

打包完成后，建议进行以下测试：

1. **在打包机器上测试**
   - 双击运行 `dist\PoetryAnswerTool.exe`
   - 检查是否能正常启动
   - 检查功能是否正常

2. **在干净的系统上测试**
   - 在没有Python环境的电脑上测试
   - 检查首次运行是否正常下载OCR模型
   - 检查所有功能是否正常

3. **检查文件大小**
   - 确认exe文件大小合理
   - 确认没有遗漏必要的文件

## 六、常见问题

### Q1: 打包失败，提示找不到模块

**解决方案**：
- 检查 `poetry_answer_tool.spec` 中的 `hiddenimports` 列表
- 确保所有本地模块都已添加

### Q2: 打包后的exe无法运行

**解决方案**：
- 检查是否被杀毒软件拦截
- 检查是否有必要的DLL文件缺失
- 尝试在命令行中运行，查看错误信息

### Q3: 打包文件太大

**解决方案**：
- 使用虚拟环境打包，避免包含不必要的包
- 排除不需要的模块
- 考虑分离OCR模型，让程序运行时下载

### Q4: 首次运行很慢

**解决方案**：
- 这是正常的，程序需要初始化OCR引擎
- 首次运行会下载OCR模型（约100MB）
- 后续运行会快很多

### Q5: 如何添加程序图标

**解决方案**：
1. 准备一个 `.ico` 格式的图标文件
2. 在 `poetry_answer_tool.spec` 文件中设置：
   ```python
   icon='icon.ico'
   ```
3. 重新打包

## 七、打包命令参考

### 完整打包命令

```bash
# 激活虚拟环境
venv\Scripts\activate

# 清理之前的打包文件
rmdir /s /q build dist

# 执行打包
pyinstaller poetry_answer_tool.spec --clean

# 或者使用批处理脚本
build.bat
```

### 打包选项说明

- `--clean`: 清理临时文件
- `--onefile`: 打包成单个exe文件（已在spec中配置）
- `--windowed`: 不显示控制台窗口（GUI程序，已在spec中配置）
- `--name`: 指定exe文件名（已在spec中配置）

## 八、版本更新

如果需要更新程序版本：

1. 修改代码
2. 测试功能
3. 重新执行打包步骤
4. 将新的exe文件分发给用户

**注意**：用户需要重新配置 `config.json`（如果配置文件格式有变化）
